## 4. ログイン仕様

1. サロンID + メールアドレスを入力
2. 最新月次タブ（`monthlySheetName(0)` で自動判定）を検索
3. 見つからなければ無料体験タブを検索
4. 見つからなければ「サロンIDが見つかりませんでした」エラー
5. メールアドレス照合（社内メアド `info@shinq-compass.jp` の場合はスキップ）
6. セッション保存（LocalStorage、有効期限あり）→ 次回アクセス時に自動ログイン

### 月次タブの自動検出
```javascript
function monthlySheetName(offsetMonths) {
  const d = new Date();
  d.setDate(1);
  d.setMonth(d.getMonth() - offsetMonths);
  const yy = String(d.getFullYear()).slice(-2);
  const mm = String(d.getMonth() + 1).padStart(2, '0');
  return yy + mm; // 例: '2603'
}
```
毎月コード変更不要。スプレッドシートに新タブを追加するだけで自動対応。

## 6. 主要機能

### 6.1 ダッシュボード
- 院名・参照シート名・表示日時・契約プランをヘッダーに表示
- プラチナPLUSプランの場合は予約リンクを常時表示

### 6.2 努力目標バナー
未取得の検索ポイントを全て取得した場合の「到達可能エリア内順位」を表示。
```
検索優先ポイント 676pt にすれば、最大エリア内順位 2位 まで可能（あと +210pt 獲得可能）
```
- 背景: 白 / 左ボーダー: 赤4px / 文字: 濃赤

### 6.3 KPI別達成状況
3ニーズ × 各指標を「達成 ✓ / 未達 ✗ / 未設定」で表示。
- 未達の場合は具体的なアクションリンクを表示
- 各ニーズの達成率をプログレスバーで可視化

### 6.4 エリア取り組み状況ヒートマップ
同じエリア内の全院の各KPI達成率を棒グラフで表示。
- データ収集: 最新月次タブ + 無料体験タブ（重複はサロンID優先で月次タブを採用）
- 自院の達成/未達を `●`/`✗` でマーク

### 6.5 口コミ投稿QRコード
しんきゅうコンパスの口コミ投稿ページへのQRコードを生成・表示・ダウンロード可能。

### 6.6 月次レポート推移（トレンドチャート）
- アクセス数・クリック率・ネット予約率・口コミ件数の月次推移を折れ線グラフで表示
- 最大2年分（24ヶ月）、3連続でデータ不在なら打ち切り
- データ収集は月次タブのみ（無料体験タブは参照しない）

### 6.7 代行設置依頼フォーム
ホームページへのバナー・QRコード設置をCS側に依頼するためのモーダルフォーム。
- 送信先: Google Apps Script → スプレッドシートに記録

### 6.8 HP設置フォーム（Googleフォーム代行）
サロンID・院名を事前入力した状態のGoogleフォームURLを生成して開く。
- フォームID: `1FAIpQLSfzJ9bcUTeSO35VK3t-iyG_2UTZWCGJ84q0Ake2l_c7LNWmtQ`

## 10. 既知の仕様・注意点

- **無料体験タブの院**: 月次タブに存在しないため、`store_pr` などのデータが `null` になる列がある → 「未設定」表示は正常動作
- **列名の揺れ**: 月次タブにより列名が変わる（例: `掲載優先ポイント` vs `検索優先ポイント`）→ `parseRowDynamic` で複数候補を持って対応
- **エリアヒートマップ**: 最新月次 + 無料体験タブの両方からエリア内院データを収集。同一サロンIDは月次タブ優先
